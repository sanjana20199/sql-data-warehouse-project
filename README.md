**Project Overview
**
** Objective**

To design and develop a modern data warehouse using SQL Server that consolidates sales data from multiple source systems and supports analytical reporting for data-driven decision-making.

⸻

**Data Engineering – Data Warehouse Development**

Key Features & Specifications
	•	Data Sources
	•	Integrated data from two source systems: ERP and CRM, provided as CSV files.
	•	Data Quality & Cleansing
	•	Handled missing values, duplicates, and inconsistent records to ensure high-quality analytical data.
	•	Data Integration
	•	Combined multiple source systems into a single, user-friendly analytical data model.
	•	Data Modeling
	•	Designed a clean schema optimized for reporting and analytical queries.
	•	Scope
	•	Focused on the latest available data (no historization or slowly changing dimensions).
	•	Documentation
	•	Included clear documentation of tables and relationships for both business users and analytics teams.

⸻
**Business Intelligence & Analytics**

Analytics Objectives

Developed SQL-based analytical queries to generate actionable insights on:
	•	Customer Behavior
	•	Purchasing patterns, repeat customers, and segmentation.
	•	Product Performance
	•	Top-selling products, revenue contribution, and demand trends.
	•	Sales Trends
	•	Overall sales growth, time-based analysis, and performance metrics.

These insights help stakeholders track key business KPIs and support strategic decision-making.
